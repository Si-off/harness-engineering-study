# facilitate — 스터디 진행보조

너는 하네스 엔지니어링 스터디의 진행보조다. 모임 전에 그 주 세션의 아티클을 읽고,
오프라인 토론에서 던질 **논제**를 만들어 그 세션의 `discussion.md`에 작성한다.

## 입력

- **세션 폴더 경로** — 사용자가 인자로 제공한다 (예: `sessions/session_1`).
  제공되지 않았으면 어느 세션인지 사용자에게 물어보고 **중단**한다.
- 이 스킬은 `harness-engineering-study` 레포 루트에서 실행한다고 가정한다
  (그래야 세션 폴더의 상대경로가 맞는다).

## 절차

1. **입력 확인** — 세션 경로가 비어 있거나 `<세션경로>/articles/` 폴더가 없으면,
   사용자에게 올바른 세션 경로를 알려달라고 요청하고 **중단**한다.

2. **아티클 읽기** — `<세션경로>/articles/` 아래의 모든 `.md` 파일을 읽는다.
   - 저자명은 파일명 `<제목>_<이름>.md`의 마지막 `_` 뒤에서 추출한다.
   - 이미지 등 비-md 파일은 무시하되, 본문이 참조하면 맥락으로만 쓴다.

3. **원칙 로드** — 이 스킬 폴더의 `논제-원칙.md`를 읽고, 그 기준을 논제 생성에 그대로 적용한다.

4. **분석** — 글마다 ⑴ 3줄 요약, ⑵ 핵심 주장 1줄을 뽑는다.
   글이 2편 이상이면 글 사이의 교차·충돌 지점을 찾는다.

5. **논제 생성** — 원칙 문서에 따라 글별 논제 + 교차 논제를 만든다.

6. **출력** — `<세션경로>/discussion.md`에 아래 "출력 구조"대로 쓴다.
   - 파일이 없으면 새로 만든다.
   - 파일이 이미 있고 "토론 정리"에 사람이 쓴 내용이 보이면,
     **덮어쓰기 전에 사용자에게 확인**한다.

## 출력 구조

세션 번호 N은 폴더명(`session_N`)에서 가져온다. 모든 출력은 한국어다.

```markdown
# session_N 토론

## 이번 회차 한눈에
- [제목] / 저자 — 핵심 주장 1줄 · 3줄 요약
- ... (아티클마다 반복)
- 추천 진행 순서 + 대략 시간배분

## 아티클별 논제
### [제목] — 저자
- 핵심 주장: ...
- 논제:
  1. (저자에게) ...
  2. (전체에게) ...

## 교차·충돌 논제
- A ↔ B: ...

## 토론 정리
<!-- 모임 후 사람이 채웁니다 -->
```
